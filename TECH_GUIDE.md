## 1. 总览：这个项目是什么？

**MediTerm Glass Pro** 是一个医学术语翻译工具，就像一本「会思考的电子词典」。

### 它能做什么？

| 功能 | 通俗解释 |
|------|----------|
| 术语搜索 | 输入中文/英文/拼音，找到对应的医学术语 |
| AI翻译 | 遇到词典里没有的词，让AI帮你翻译 |
| 批量翻译 | 一次翻译一大段文字 |
| 个人词典 | 添加自己常用的术语，下次直接查 |
| 收藏夹 | 把常用的词收藏起来，方便查看 |

## 2. 核心技术解释

### 2.1 React — 页面的「积木」

**一句话解释**：React 是用来搭建网页的工具，就像用乐高积木搭房子。

**通俗比喻**：
- 想象你在装修房子，每个房间（组件）可以单独设计
- 卧室、厨房、客厅都是独立的「组件」
- 改动卧室不会影响厨房
- React 就是帮你管理这些「房间」的工具

```
┌─────────────────────────────────────┐
│              App（整个房子）          │
│  ┌─────────┐  ┌─────────┐          │
│  │ 搜索框  │  │ 结果列表 │          │
│  │（组件1）│  │（组件2） │          │
│  └─────────┘  └─────────┘          │
│  ┌───────────────────────┐         │
│  │      翻译助手（组件3） │         │
│  └───────────────────────┘         │
└─────────────────────────────────────┘
```

### 2.2 TypeScript — 更安全的 JavaScript

**一句话解释**：TypeScript 是 JavaScript 的「升级版」，能在写代码时发现错误。

**通俗比喻**：
- JavaScript 就像写信不检查错别字，直接寄出去
- TypeScript 就像有个助手，写完帮你检查一遍再寄
- 错误在「寄出前」就能发现，不用等对方收到后才说「看不懂」

| 对比 | JavaScript | TypeScript |
|------|------------|------------|
| 比喻 | 裸奔 | 穿防护服 |
| 错误发现时机 | 运行时（太晚了） | 编写时（及时发现） |
| 代码提示 | 较少 | 很智能 |
| 学习难度 | 简单 | 稍难一点 |

### 2.3 Vite — 超快的「厨房」

**一句话解释**：Vite 是开发工具，让你写代码时网页秒刷新。

**通俗比喻**：
- 传统工具（Webpack）：做菜前要把所有食材都切好、备好，再开始炒
- Vite：边切边炒，客人点什么就先做什么，超级快

| 对比 | 传统工具 | Vite |
|------|---------|------|
| 启动速度 | 像煮火锅（慢） | 像微波炉（秒热） |
| 修改后刷新 | 等几秒 | 几乎瞬间 |
| 原理 | 打包所有文件 | 按需加载 |

### 2.4 Zustand — 数据的「中央仓库」

**一句话解释**：Zustand 管理整个应用的数据状态，所有组件共享同一份数据。

**通俗比喻**：
- 想象一个大家庭，大家都把钱放在一个保险箱里（Zustand）
- 需要钱时去保险箱拿，而不是每个人各存各的
- 这样每个人都知道家里还剩多少钱

```
┌─────────────────────────────────────┐
│         Zustand 数据仓库             │
│  ┌─────────────────────────────┐   │
│  │ 用户词典: [{术语1}, {术语2}] │   │
│  │ 收藏列表: ["id1", "id2"]    │   │
│  │ 搜索历史: ["感冒", "发烧"]  │   │
│  │ 设置项:   {暗色模式: false} │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
         ↑    ↑    ↑    ↑
    组件A  组件B  组件C  组件D（都能读写）
```

### 2.5 Fuse.js — 智能模糊搜索

**一句话解释**：Fuse.js 让你打错字也能找到想要的结果。

**通俗比喻**：
- 普通搜索：你输入「感帽」，它说「找不到」
- Fuse.js：你输入「感帽」，它猜你可能想找「感冒」，帮你找出来

| 搜索方式 | 输入 | 普通搜索结果 | Fuse.js 结果 |
|---------|------|-------------|--------------|
| 错别字 | 感帽 | 找不到 | 感冒 ✓ |
| 拼音 | ganmao | 找不到 | 感冒 ✓ |
| 部分匹配 | 冒 | 找不到 | 感冒 ✓ |

### 2.6 Pinyin-pro — 拼音转换器

**一句话解释**：把中文变成拼音，让用户可以用拼音搜索。

**通俗比喻**：
- 就像给每个中文字「起个英文外号」
- 输入「gm」（感冒拼音首字母）就能找到「感冒」

```
输入：感冒
     ↓
Pinyin-pro 处理
     ↓
输出：
  - 完整拼音：gǎn mào
  - 首字母：gm
```

### 2.7 Tailwind CSS — 快速穿衣服

**一句话解释**：Tailwind 是写样式的工具，用简短的「标签」让网页变漂亮。

**通俗比喻**：
- 传统 CSS：自己设计衣服，从零开始
- Tailwind：去商场挑标签（如「红色」「大号」「圆角」），直接穿

```html
<!-- 传统写法：写一堆样式 -->
<button style="background: blue; padding: 10px; border-radius: 5px;">
  按钮
</button>

<!-- Tailwind 写法：贴标签 -->
<button class="bg-blue-500 p-2 rounded">
  按钮
</button>
```

| 样式需求 | Tailwind 标签 | 含义 |
|---------|---------------|------|
| 蓝色背景 | `bg-blue-500` | background: blue |
| 内边距 | `p-4` | padding: 1rem |
| 圆角 | `rounded` | border-radius |
| 阴影 | `shadow-lg` | 大阴影 |
| 居中 | `flex justify-center` | 弹性布局居中 |

### 2.8 Google Gemini API — AI 大脑

**一句话解释**：连接谷歌的 AI，让它帮你翻译词典里没有的医学术语。

**通俗比喻**：
- 本地词典是「字典」，有限
- Gemini AI 是「专家顾问」，问什么都能答

```
用户输入 "Hypertrophic cardiomyopathy"
        ↓
本地词典查询 → 没找到
        ↓
发送给 Gemini AI
        ↓
AI 返回："肥厚型心肌病"
```

### 2.9 XLSX — Excel 读写工具

**一句话解释**：让网页能读写 Excel 文件。

**通俗比喻**：
- 你可以把词典导出成 Excel，在电脑上编辑
- 编辑完再导入回来，词典就更新了

```
┌─────────────┐        ┌─────────────┐
│   网页词典   │ ←导入→ │  Excel文件  │
│ (在浏览器)  │ ←导出→ │ (在电脑上)  │
└─────────────┘        └─────────────┘
```

### 2.10 Web Speech API — 让网页说话

**一句话解释**：浏览器自带的功能，能把文字读出来。

**通俗比喻**：
- 就像手机的「朗读」功能
- 点击英文术语旁边的喇叭图标，就能听发音

## 4. 数据如何流动？

### 4.1 搜索一个术语的流程

```
┌─────────────────────────────────────────────────────────────┐
│                    用户输入 "感冒"                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤1: Pinyin-pro 把「感冒」转成拼音「ganmao」和「gm」        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤2: Fuse.js 在本地词典里搜索                              │
│        - 先找精确匹配                                        │
│        - 再找拼音匹配                                        │
│        - 最后模糊匹配                                        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤3: 找到结果！返回：                                       │
│        中文: 感冒                                            │
│        英文: Common Cold                                     │
│        分类: 内科                                            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤4: React 组件显示结果到页面上                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 AI翻译的流程

```
用户输入一段医学文本
        ↓
点击「翻译」按钮
        ↓
┌────────────────────────┐
│ 1. 检查本地词典有没有   │
│    → 有就直接用        │
│    → 没有继续往下      │
└────────────────────────┘
        ↓
┌────────────────────────┐
│ 2. 调用 Gemini API     │
│    发送: 请翻译这段文字 │
│    等待 AI 返回结果    │
└────────────────────────┘
        ↓
┌────────────────────────┐
│ 3. 显示翻译结果        │
│    用户可以复制/收藏   │
└────────────────────────┘
```

### 4.3 数据存储位置

| 数据类型 | 存在哪里 | 会丢失吗 |
|---------|---------|---------|
| 用户词典 | 浏览器 localStorage | 清除浏览器数据会丢失 |
| 收藏列表 | 浏览器 localStorage | 清除浏览器数据会丢失 |
| 搜索历史 | 浏览器 localStorage | 清除浏览器数据会丢失 |
| 设置选项 | 浏览器 localStorage | 清除浏览器数据会丢失 |
| 系统词典 | 代码文件里 | 不会丢失 |

> **小提示**：记得定期用「导出」功能备份你的用户词典！

## 5. 常见问题解答

### Q1: 为什么用 React 而不是 Vue？

两者都很好，选择 React 是因为：
- 社区更大，资源更多
- 和 TypeScript 配合更好
- 组件化思想更纯粹

### Q2: TypeScript 是不是更难学？

稍微难一点，但好处很大：
- 写代码时就能发现错误
- 编辑器会智能提示
- 长期来看省时间

### Q3: 数据会上传到服务器吗？

**不会！** 所有数据都存在你自己的浏览器里。只有使用 AI 翻译时，翻译内容会发送给 AI（Gemini），但不会保存。

### Q4: 为什么需要 API Key？

API Key 就像「会员卡」：
- 使用 AI 翻译需要连接 Google Gemini
- Google 需要知道谁在用他们的服务
- 所以需要一个「身份证明」

### Q5: 项目能离线用吗？

**部分功能可以：**
- ✅ 本地词典搜索（离线可用）
- ✅ 收藏、历史查看（离线可用）
- ❌ AI 翻译（需要联网）
